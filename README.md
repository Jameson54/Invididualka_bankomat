Симуляция банкомата (ATM Simulation)

Индивидуальный проект — паттерн проектирования Singleton.

Автор: Кондратенко Павел  
Группа: ПИ-251


Постановка задачи

Необходимо разработать программу имитации банкомата, которая позволяет пользователям:

- Вносить деньги на счёт
- Снимать деньги со счёта
- Проверять текущий баланс
- Просматривать историю операций

Реализация: консольное приложение на C# (.NET Framework / .NET Core).


Реальный сценарий применения

Банкомат — это объект, который должен существовать в единственном экземпляре в системе. В реальной жизни нельзя иметь два банкомата с разными балансами. Поэтому используется паттерн Singleton, который гарантирует, что будет создан только один экземпляр банкомата.

Решение:
1. Один экземпляр банкомата создаётся при первом обращении.
2. Все операции (внесение, снятие, проверка) проходят через этот единственный экземпляр.
3. Баланс и история операций сохраняются в файлы (balance.txt, history.txt).
4. При перезапуске программы данные восстанавливаются из файлов.

Так сокращается время на рутинные операции и снижается риск ошибок при копировании данных.


Паттерн Singleton

Зачем нужен
Создание нескольких экземпляров банкомата с разными балансами не имеет смысла. Singleton позволяет иметь только один объект банкомата, доступный из любой части программы.

Какую проблему решает в проекте

| Проблема | Решение через Singleton |
|----------|------------------------|
| Создание нескольких банкоматов с разными балансами | Приватный конструктор запрещает создание через `new ATM()` |
| Потеря данных после перезапуска | Сохранение баланса и истории в файлы |
| Разные части программы имеют доступ к разным экземплярам | Статическое свойство `Instance` возвращает один и тот же объект |

Ключевые элементы

- ATM - конкретный прототип (Singleton).
- FileStorage - сохранение баланса и истории в файлы.
- Transaction - модель операции (пополнение, снятие).
- Menu - пользовательский интерфейс.

Использование

После запуска отображается меню:


=== ATM SIMULATION ===
Welcome!

Select an action:
1. Deposit money
2. Withdraw money
3. Check balance
4. Transaction history
5. Exit
Your choice:


Пункт 1 — Внести деньги

1. Ввести сумму для внесения.
2. Программа проверяет, что сумма больше нуля.
3. Баланс увеличивается, операция сохраняется в истории.

Пункт 2 — Снять деньги

1. Ввести сумму для снятия.
2. Программа проверяет, что сумма больше нуля и не превышает баланс.
3. Баланс уменьшается, операция сохраняется в истории.

Пункт 3 — Проверить баланс

Показывает текущий баланс.

Пункт 4 — История операций

Показывает все операции (дата, время, тип, сумма, баланс после операции).

Пункт 5 — Выход

Завершает программу.

История разработки (GitHub Flow)

Разработка велась по этапам в отдельных ветках с коммитами:

1. Сборка каркаса (Transaction, ATM, FileStorage)
2. Паттерн Singleton (приватный конструктор, Instance)
3. Меню и ввод (Menu.cs)
4. Сохранение в файлы (FileStorage.cs)
5. Демо-сценарии (тестирование)
6. Рефакторинг (исправление стиля, удаление магических чисел)

Примечания

- Программа сохраняет баланс и историю операций в файлы `balance.txt` и `history.txt`.

- При вводе букв вместо цифр программа запрашивает повторный ввод.
